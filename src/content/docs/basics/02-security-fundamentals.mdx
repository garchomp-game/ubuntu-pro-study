## この章で学ぶこと

Ubuntu Proの各サービスを理解する前に、以下の基礎概念を押さえましょう。

- **CVE**（共通脆弱性識別子）
- **Mainリポジトリ と Universeリポジトリ**
- **セキュリティパッチの仕組み**
- **Ubuntu LTSのサポートライフサイクル**

---

## CVE（Common Vulnerabilities and Exposures）

### CVEとは？

**CVE** は、ソフトウェアの脆弱性（セキュリティの穴）に付けられる **世界共通の識別番号** です。

```
例: CVE-2024-1234
     ↑     ↑    ↑
     共通名  年   連番
```

例えるなら、「医療でいう病名」のようなものです。世界中のセキュリティ専門家が同じ脆弱性について話すとき、CVE番号を使うことで誤解なく通じ合えます。

### CVSSスコア（深刻度）

各CVEには**CVSS（Common Vulnerability Scoring System）** という深刻度スコアが付きます。

| 深刻度 | CVSSスコア | 説明 |
|--------|:---:|------|
| **Critical** | 9.0〜10.0 | 即座に悪用可能。最優先で対処 |
| **High** | 7.0〜8.9 | 重大な影響。早急に対処 |
| **Medium** | 4.0〜6.9 | 条件付きで影響。計画的に対処 |
| **Low** | 0.1〜3.9 | 影響は限定的 |

:::caution[Canonicalの対応方針]
Ubuntu Proでは、**Critical・High** の脆弱性に対して迅速なパッチ提供を約束しています。Criticalな脆弱性は **24時間以内** の修正を目標としています。
:::

## Ubuntuのリポジトリ構造

### Mainリポジトリ

Ubuntuの**中核パッケージ**が含まれるリポジトリです。

- Canonical（Ubuntu開発元）が **直接メンテナンス** する
- 約 **2,300パッケージ** / リリースごと
- OSの基本機能、カーネル、デスクトップ環境のコア部分など
- 通常のUbuntu LTSでは **5年間** セキュリティサポートされる

**含まれるパッケージの例:**
```
linux-kernel, systemd, bash, apt, openssl, python3, gcc, firefox, ...
```

### Universeリポジトリ

**コミュニティが管理する広大なパッケージ群** です。

- Debianやコミュニティからの **膨大なパッケージ集**
- 約 **36,000パッケージ以上** / リリースごと
- 開発ツール、ライブラリ、アプリケーションなど
- 通常のUbuntuでは **Canonicalによるセキュリティ保証はなし**

**含まれるパッケージの例:**
```
nodejs, php, ruby, redis, nginx, postgresql, rust, go, ...
```

### MainとUniverseの違いを図で理解

```
Ubuntu パッケージの全体像:
┌──────────────────────────────────────────────────┐
│                  Ubuntu Archive                   │
│  ┌──────────────┐  ┌──────────────────────────┐  │
│  │    Main      │  │       Universe           │  │
│  │  ~2,300 pkg  │  │     ~36,000+ pkg         │  │
│  │              │  │                          │  │
│  │  Canonical   │  │ コミュニティ管理          │  │
│  │  が保守      │  │ 通常: 保証なし           │  │
│  │              │  │ Pro: Canonicalが保守 ✅   │  │
│  │  5年保証     │  │                          │  │
│  │  Pro: 10年   │  │ Pro: 10年保証            │  │
│  └──────────────┘  └──────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

:::note[これが重要]
Ubuntu Proの最大の価値の一つは、**Universe リポジトリの 36,000以上のパッケージにもセキュリティ保証が付く**ことです。開発者が日常的に使う Node.js、Python ライブラリ、PHP、Redisなどが含まれます。
:::

## セキュリティパッチの仕組み

### パッチが適用されるまでの流れ

```
1. 脆弱性の発見
   └─ セキュリティ研究者、開発者、自動スキャンなどで発見

2. CVE番号の発行
   └─ MITRE/NISTがCVE IDを割り当て、公開

3. Canonicalのセキュリティチームが評価
   └─ Ubuntu環境での影響度を判定
   └─ Ubuntu Priorityを割り当て（Critical/High/Medium/Low）

4. パッチの開発・テスト
   └─ 修正を開発し、リグレッションテストを実施

5. パッチの配布
   └─ apt update && apt upgrade で適用可能に
   └─ unattended-upgrades で自動適用も可能
```

### 自動更新（unattended-upgrades）

Ubuntuにはデフォルトで `unattended-upgrades` がインストールされており、**セキュリティパッチを自動的に適用** します。

```bash
# 自動更新の設定確認
cat /etc/apt/apt.conf.d/20auto-upgrades

# 期待される出力:
APT::Periodic::Update-Package-Lists "1";
APT::Periodic::Unattended-Upgrade "1";
```

## Ubuntu LTSのサポートライフサイクル

### 通常のサポート期間

```
                    Standard Support (5年)
Ubuntu 20.04  ├──────────────────────────┤
              2020                       2025

                    Standard Support (5年)
Ubuntu 22.04  ├──────────────────────────┤
              2022                       2027

                    Standard Support (5年)
Ubuntu 24.04  ├──────────────────────────┤
              2024                       2029
```

### Ubuntu Pro適用時

```
              Standard (5年)    ESM (5年)     Legacy (5年)
Ubuntu 24.04  ├────────────────┼─────────────┼────────────┤
              2024             2029          2034         2039
              └─── 無料 ───┘  └── Pro ──┘   └─ 有料 ─┘
                                (無料で利用可)
```

### サポート終了（EOL）後のリスク

サポートが終了したUbuntuを使い続けるリスク:

1. **既知の脆弱性が修正されない** → 攻撃者に狙われやすくなる
2. **新しいソフトウェアがインストールしにくくなる** → 依存関係が壊れる
3. **コンプライアンス違反** → 企業利用では監査に通らない

:::tip[例: Ubuntu 24.04 LTSの場合]
Ubuntu 24.04 LTSでesm-infra と esm-apps を有効化すると、**2034年まで** セキュリティサポートが保証されます。
:::

## まとめ

| 概念 | 説明 |
|------|------|
| **CVE** | ソフトウェアの脆弱性に付けられる世界共通の識別番号 |
| **CVSS** | 脆弱性の深刻度を0〜10で数値化したスコア |
| **Main** | Canonical管理の中核パッケージ（〜2,300個）。5年保証 |
| **Universe** | コミュニティ管理の広大なパッケージ群（〜36,000+個） |
| **ESM** | 5年を超えるセキュリティメンテナンスの延長（Proで利用可） |
| **unattended-upgrades** | セキュリティパッチの自動適用機能 |

次の章では、Ubuntu Proの中核サービスである **ESM（拡張セキュリティメンテナンス）** を詳しく解説します。
