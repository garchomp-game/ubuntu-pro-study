## FIPSとは

**FIPS 140（Federal Information Processing Standard 140）** は、**米国政府が定めた暗号モジュールのセキュリティ規格**です。

```bash
fips-updates     yes       disabled     FIPS compliant crypto packages with stable security updates
```

家庭用途では **disabled（無効）** のままで問題ありません。

---

## なぜFIPSが存在するのか

### 背景

```
暗号化の問題:
  「暗号化しています」と言っても…

  ❓ どんなアルゴリズムを使っている？
  ❓ その実装は正しい？
  ❓ 脆弱な暗号化方式ではない？
  ❓ 誰がそれを検証した？

FIPS 140 の答え:
  ✅ 政府認定の検証機関が暗号モジュールをテスト
  ✅ 合格したものだけに「FIPS認証」を付与
  ✅ 政府機関はFIPS認証済みの暗号のみ使用可能
```

### 対象となる暗号モジュール

Ubuntu Proで認証されている主なモジュール:
- **OpenSSL** — Webサーバー、TLS通信の暗号化
- **Libgcrypt** — GnuPGなどの暗号処理
- **GnuTLS** — TLS通信ライブラリ
- **Linux Kernel Crypto API** — カーネルレベルの暗号処理

---

## fips と fips-updates の違い

Ubuntu Proには2つのFIPS関連サービスがあります。

### fips（厳密認証モード）

```
特徴:
- NIST（米国標準技術研究所）が正式に認証したバージョン
- そのバージョンが「固定」される
- 新しいセキュリティパッチは即座には適用されない

メリット: 厳密なコンプライアンスを満たせる
デメリット: 新しい脆弱性に対応が遅れる
```

### fips-updates（推奨モード）

```
特徴:
- FIPS認証済みモジュールに、セキュリティパッチを追加
- 脆弱性が見つかれば素早く修正される
- Canonicalが定期的にNISTへ再認証を申請

メリット: コンプライアンスとセキュリティの両立
デメリット: 厳密には「認証中」のバージョンになる場合がある
```

:::tip[推奨]
FIPSが必要な場合は、`fips-updates` が推奨されます。セキュリティパッチを受けられるため、厳密な`fips`よりも安全です。
:::

## FIPSが必要な人とは

### 必要な場合

| ケース | 例 |
|--------|-----|
| **米国政府機関** | 連邦機関のシステム |
| **米国政府の請負業者** | 政府向けソフトウェアを開発する企業 |
| **金融機関** | 規制要件としてFIPS準拠が求められる |
| **医療機関** | HIPAA準拠の一環としてFIPSが要求される |
| **FedRAMP認証** | クラウドサービスの政府向け認証 |

### 必要ではない場合

| ケース | 理由 |
|--------|------|
| **家庭用PC** | 政府のコンプライアンス要件がない |
| **個人の開発環境** | 規制対象ではない |
| **日本国内の一般企業** | 日本にはFIPS準拠の法的義務はない（場合が多い） |

## 家庭ユーザーが知るべきこと

### FIPSは有効にすべき？

**いいえ、家庭ユーザーはFIPSを有効にする必要はありません。**

理由:
1. 政府向けのコンプライアンス規格であり、家庭では要件がない
2. FIPSを有効にすると一部のソフトウェアの動作に影響が出る可能性がある
3. Ubuntu標準の暗号化（OpenSSLなど）でも十分に安全

### でも知識としては重要

FIPSを理解していると:
- 「暗号化の信頼性」がどう検証されているかがわかる
- エンタープライズ環境の要件を理解できる
- セキュリティ関連の記事やニュースの理解が深まる

## FIPS と「普通の暗号化」の関係

```
普通のUbuntuの暗号化:
┌──────────────────────────────────────┐
│  OpenSSL (標準版)                      │
│  ・最新のセキュリティパッチ適用済み      │
│  ・十分に安全                          │
│  ・FIPS「未認証」だが、同じ暗号を使用    │
└──────────────────────────────────────┘

FIPS有効時のUbuntuの暗号化:
┌──────────────────────────────────────┐
│  OpenSSL (FIPS認証版)                  │
│  ・NISTが公式に認証済み                 │
│  ・「安全である」と政府が証明            │
│  ・コンプライアンス監査に合格できる       │
└──────────────────────────────────────┘

実態: どちらも同じ暗号アルゴリズム（AES, RSAなど）を使用
違い: 「公式に認証された実装かどうか」
```
