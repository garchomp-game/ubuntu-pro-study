## この章で扱うサービス

`pro status` に表示される残り3つのサービスを解説します。

```bash
anbox-cloud      yes       disabled     Scalable Android in the cloud
realtime-kernel* yes       disabled     Ubuntu kernel with PREEMPT_RT patches integrated
landscape        yes       disabled     Management and administration tool for Ubuntu
```

いずれも家庭ユーザーが有効にする必要はありませんが、**何者なのかを理解しておく**ことは有益です。

---

## Anbox Cloud

### 概要

**Anbox Cloud** は、クラウドサーバー上で **Androidアプリケーションを大規模に実行する** ためのプラットフォームです。

```
Anbox Cloud のイメージ:

┌─────────── クラウドサーバー ────────────┐
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐     │
│  │ And │ │ And │ │ And │ │ And │ ... │
│  │roid │ │roid │ │roid │ │roid │     │
│  │ #1  │ │ #2  │ │ #3  │ │ #4  │     │
│  └─────┘ └─────┘ └─────┘ └─────┘     │
│        LXD コンテナ / 仮想マシン        │
│              Ubuntu Server              │
└─────────────────────────────────────────┘
          │           │           │
          ▼           ▼           ▼
    ユーザーA    ユーザーB    ユーザーC
    (ストリーミングで利用)
```

### 主な用途

| 用途 | 説明 |
|------|------|
| **モバイルアプリテスト** | 多数のAndroid環境で同時テスト |
| **クラウドゲーミング** | Androidゲームを低遅延でストリーミング |
| **自動車産業** | Android Automotive OSの開発 |
| **業務自動化** | Androidアプリを使ったワークフロー自動化 |

### 家庭ユーザーとの関係

:::note[デスクトップ向けではない]
Anbox Cloud は **サーバー/クラウド専用** であり、Ubuntu Desktopへのインストールは**サポートされていません**。かつて存在した「Anbox」（デスクトップでAndroidアプリを動かすプロジェクト）とは別物です。

家庭ユーザーが有効にする理由はありません。
:::

## Realtime Kernel（リアルタイムカーネル）

### 概要

**Realtime Kernel** は、**PREEMPT_RT パッチ** が適用された特殊なLinuxカーネルです。通常のカーネルが「スループット（処理量）」を重視するのに対し、リアルタイムカーネルは「レイテンシ（応答速度）」を重視します。

```
通常のカーネル:
  タスクA ████████░░░░████████░░░░
  タスクB ░░░░████████░░░░████████
  → 全体の処理量は最大化するが、各タスクの応答時間はバラつく

リアルタイムカーネル:
  高優先度  ████░████░████░████░████
  低優先度  ░░░░█░░░░█░░░░█░░░░█░░░
  → 高優先度タスクが確実に期限内に実行される
```

### PREEMPT_RT のキーテクノロジー

| 機能 | 説明 |
|------|------|
| **完全カーネルプリエンプション** | カーネルコードのほぼすべてを割り込み可能に |
| **スレッド化割り込み** | ハードウェア割り込みをスレッドとして扱い、優先度制御可能に |
| **優先度継承** | 優先度逆転問題を自動解決 |
| **決定論的スケジューリング** | タスク切り替えを保証時間内に実行 |

### 主な用途

```
リアルタイムカーネルが必要な場面:

🏭 産業制御 — ロボットアーム、CNCマシン、PLCの制御
  → ミリ秒単位の遅延がエラーや事故につながる

🏥 医療機器 — 生命維持装置、患者モニタリング
  → 応答遅延は人命に関わる

🚗 自動車 — ABS、自動運転支援システム
  → リアルタイム応答が安全性の基盤

🎵 プロオーディオ — 低レイテンシ音声処理
  → 音の遅延はライブパフォーマンスに致命的

📡 通信 — 5G基地局、低遅延ネットワーク
  → マイクロ秒レベルの応答が必要
```

### 家庭ユーザーとの関係

:::tip[DTM/音楽制作をする場合]
もしあなたがLinux上で **DTM（デスクトップミュージック）** や **プロオーディオ制作** を行う場合、リアルタイムカーネルは有用です。JACK Audio Connection Kit と組み合わせることで、超低レイテンシの音声処理が可能になります。

それ以外の一般的なデスクトップ用途では、リアルタイムカーネルのメリットはほとんどありません。むしろ全体的なスループットが低下する可能性があります。
:::

## Landscape

### 概要

**Landscape** は、Canonicalが提供する **Ubuntu の大規模管理ツール** です。

```
Landscape のイメージ:

       ┌─────────────────────────────┐
       │     Landscape ダッシュボード    │
       │  ┌─────────────────────────┐ │
       │  │ PC #1  ✅ 最新          │ │
       │  │ PC #2  ⚠️ 更新あり      │ │
       │  │ Server #1  ✅ 最新      │ │
       │  │ Server #2  ❌ 要対応    │ │
       │  │ ...                     │ │
       │  └─────────────────────────┘ │
       └─────────────────────────────┘
           │        │        │
           ▼        ▼        ▼
         PC #1   PC #2   Server #1  ...
```

### 主な機能

| 機能 | 説明 |
|------|------|
| **パッケージ管理** | 全マシンのパッケージ更新を一元管理 |
| **段階的デプロイ** | 更新をグループごとに段階適用 |
| **インベントリ管理** | ハードウェア・ソフトウェアの資産台帳 |
| **セキュリティ監査** | 各マシンのセキュリティ状態を可視化 |
| **コンプライアンス** | ハードニングの適用状態を監視 |
| **リモート操作** | コマンドの一斉実行 |

### 家庭ユーザーとの関係

:::note
Landscapeは **10台、100台、1000台…** のUbuntuマシンを管理するためのツールです。1〜2台の家庭用PCには**オーバースペック**です。

ただし、もしあなたが将来的にサーバー管理やインフラエンジニアの仕事に興味があるなら、Landscapeの概念（大規模なマシン管理の考え方）は理解しておくと有益です。
:::

## 3サービスのまとめ

| サービス | 対象ユーザー | 家庭用途 | 一言まとめ |
|----------|:---:|:---:|------|
| **Anbox Cloud** | 🏢 企業 | ❌ 不要 | クラウドでAndroidを大規模実行 |
| **Realtime Kernel** | 🏭 産業/🎵 音楽 | △ 限定的 | 応答時間を保証する特殊カーネル |
| **Landscape** | 🏢 企業 | ❌ 不要 | Ubuntuの大規模管理ツール |

これら3つはすべて **disabled（無効）** のままで問題ありません。
