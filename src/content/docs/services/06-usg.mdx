## USGとは

**USG（Ubuntu Security Guide）** は、Ubuntuシステムのセキュリティ設定を**業界標準のベンチマークに照らして監査・修正する**ための自動化ツールです。

```bash
usg              yes       disabled     Security compliance and audit tools
```

家庭用途では通常 **disabled（無効）** のままで問題ありません。

---

## USGが解決する問題

### セキュリティ設定の難しさ

```
OSのセキュリティ設定は膨大:

✗ SSHの設定は安全か？
✗ パスワードポリシーは適切か？
✗ 不要なサービスは無効化されているか？
✗ ファイルのパーミッションは正しいか？
✗ 監査ログは有効か？
✗ カーネルパラメータは安全な値か？
  ... 何百もの設定項目がある

→ 人手で全部チェックするのは非現実的
→ USGが自動で監査・修正してくれる
```

## CIS Benchmarks と DISA-STIG

USGが準拠する2つの主要なセキュリティ基準を理解しましょう。

### CIS Benchmarks

**CIS（Center for Internet Security）** が策定するセキュリティ設定のベストプラクティス集です。

| 項目 | 内容 |
|------|------|
| **策定者** | CIS（非営利団体） |
| **対象** | あらゆる組織 |
| **内容** | 何百もの設定推奨事項 |
| **レベル** | Level 1（基本） / Level 2（強化） |
| **例** | パスワード最小長、SSH設定、ファイアウォール設定など |

CIS Benchmarkの設定項目の例:

```
[CIS 1.1.1.1] cramfsファイルシステムの無効化
  → cramfsは古い圧縮ファイルシステム。攻撃に使われる可能性がある

[CIS 5.2.1] SSHプロトコルのバージョン確認
  → SSH v1 は脆弱。v2 のみを許可すべき

[CIS 5.3.1] パスワードの最小長
  → 8文字以上を要求すべき

[CIS 1.4.1] GRUBブートローダーのパスワード設定
  → 物理アクセスからの保護
```

### DISA-STIG

**DISA（Defense Information Systems Agency）** が策定する **米国国防総省のセキュリティ設定ガイド** です。

| 項目 | 内容 |
|------|------|
| **策定者** | DISA（米国国防情報システム局） |
| **対象** | 米国国防総省関連システム |
| **厳格さ** | CISより厳しい |
| **カテゴリ** | CAT I（高）/ CAT II（中）/ CAT III（低） |

:::note
DISA-STIGはCIS Benchmarksよりも厳格で、主に軍事・国防関連のシステムで要求されます。民間企業でも、米国政府と取引がある場合に必要になることがあります。
:::

## USGの3つの機能

### 1. 監査（Audit）

システムの現在の設定を指定されたベンチマークと照合し、適合・不適合をレポートします。

```bash
# CIS Level 1 で監査
sudo usg audit cis_level1_server

# 出力: HTML/XML形式のレポート
# 各ルールのPass/Failが一覧で表示される
```

### 2. 修正（Fix）

不適合な設定を自動的に修正します。

```bash
# CIS Level 1 に準拠するよう自動修正
sudo usg fix cis_level1_server

# 注意: 新規インストールのシステムで実行推奨
```

:::caution
`usg fix` は多くのシステム設定を変更します。本番環境や日常使用のPCでいきなり実行すると、**正常に動作しなくなる可能性**があります。テスト環境で試してから適用しましょう。
:::

### 3. カスタマイズ（Customize）

すべてのルールを一律に適用するのではなく、環境に合わせて取捨選択できます。

```bash
# テーラリングファイルでカスタマイズ
# 特定のルールを無効化したり、値を変更したりできる
```

## USGの技術的バックエンド

USGは内部で **OpenSCAP** というオープンソースのセキュリティ監査ツールを使用しています。

```
USG の仕組み:

USG コマンド
    │
    ▼
OpenSCAP エンジン
    │
    ▼
SCAP コンテンツ（CIS / STIG のルール定義）
    │
    ▼
システムの設定を評価 → レポート生成
```

**SCAP（Security Content Automation Protocol）** は、セキュリティ設定のチェックを自動化するための標準規格です。

## 家庭ユーザーにとっての意味

### USGは有効にすべき？

**通常は不要です。** 理由:

1. CIS/STIGは企業・政府向けのコンプライアンス基準
2. 家庭用PCではこれらの監査要件がない
3. USGの修正は厳格すぎて、日常利用に支障が出る可能性がある
   - 例: パスワードポリシーの厳格化、一部サービスの無効化

### でも学習目的では面白い

```bash
# もし試してみたい場合（監査だけなら安全）:

# USGのインストール
sudo pro enable usg

# CIS Level 1 で監査（読み取りのみ、変更なし）
sudo usg audit cis_level1_workstation

# → レポートで「PCがどれだけセキュリティ基準に適合しているか」がわかる
```

:::tip
`usg audit` は**読み取り専用**なので、システムに変更を加えません。セキュリティ学習として、自分のPCがどの程度基準を満たしているか確認するのは面白い実験です。
:::
